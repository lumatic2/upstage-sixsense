# 리서치 — 예산 기반 음식 추천/학식 정보 서비스 시장 사례와 차별화 여지

> 접근일: 2026-07-19 (모든 URL 이하 동일 접근일, 개별 표기 생략). 도구: WebSearch, ncli(Naver blog/news).
> 목적: 데모데이 발표에서 "그냥 ChatGPT에 물어보면 되지 않나?" / "거지맵이랑 뭐가 다른가?"에 쓸 근거 확보.
> 선행 리서치와의 관계: `research/2026-07-15-*.md` 4건(특히 `product-conflict-gate.md`, `source-feasibility.md`)이 거지맵·학식샤맵·EAT-SSU·에브리타임·카카오맵 등 주요 경쟁 매트릭스를 이미 확정했다. 이 문서는 그 위에 **① 캠퍼스픽 실사(오해 정정) ② 메뉴판 사진→구조화 데이터 사례 유무 ③ ChatGPT 한계의 구체 사례 ④ 명륜동/혜화 식당 정보의 공개 소스 현황**을 신규로 채운다. 중복 재조사는 하지 않았다.

## 1. 국내외 유사 서비스 재확인 + 신규 항목

### 1-1. 캠퍼스픽 — 오해 정정 (신규 실사)

과업 지시에 "학식 알림 앱(에브리타임 학식, 캠퍼스픽 등)"으로 캠퍼스픽이 언급됐으나, 실사 결과 **캠퍼스픽은 학식/식당 서비스가 아니다.**

- 캠퍼스픽(운영사 VINU, 현재 "에브리커리어")은 대학생 커뮤니티·동아리·공모전·대외활동·취업 정보 플랫폼이다. 식단·식당 기능 없음.
- 출처: [캠퍼스픽 앱 - App Store](https://apps.apple.com/kr/app/%EC%BA%A0%ED%8D%BC%EC%8A%A4%ED%94%BD/id1193606934), [캠퍼스픽 - 나무위키](https://namu.wiki/w/%EC%BA%A0%ED%8D%BC%EC%8A%A4%ED%94%BD), [에브리커리어(캠퍼스픽) 공식](https://www.campuspick.com/) — 접근 2026-07-19.
- 발표에서 캠퍼스픽을 경쟁/대체재로 언급하지 않는 편이 안전하다(사실관계 오류 리스크).

### 1-2. 학식 알림 앱 — 확인된 실사례 (신규 보강)

기존 `product-conflict-gate.md`가 학식샤맵(서울대)·EAT-SSU(숭실대)·에브리타임을 확인했다. 이번 조사로 **서울대 "식샤(Siksha)"** 앱을 추가로 확인했다 — 와플스튜디오(서울대 학생 개발 동아리) 제작, 대학원 기숙사식당·85동 수의대식당·301동 테이크아웃 메뉴 제공, 메뉴별 별점·즐겨찾기 기능.
- 출처: [식샤: 서울대학교 학식 알리미 - Google Play](https://play.google.com/store/apps/details?id=com.wafflestudio.siksha2&hl=en_US), [식샤 공식](https://siksha.wafflestudio.com/) — 접근 2026-07-19.
- 시사점: "학식 전용 앱"은 대학마다 학생 사이드프로젝트로 이미 여러 번 만들어졌다(학식샤맵·식샤·EAT-SSU와 동일 패턴) — `product-conflict-gate.md` §5의 "진입장벽 없음" 판정을 재확인.
- 성균관대 자체 학식 전용 서드파티 앱은 이번 검색에서 발견되지 않음(공식 KINGO-M 앱만 확인, §1-4).

### 1-3. 예산·가성비 식당 추천 — 거지맵 재확인, 신규 URL 없음

`product-conflict-gate.md`가 이미 확정: 거지맵(geojimap.com)이 예산 필터(1,000~8,000원)+학식+대학가 지도를 전국 단위로 제공, 누적 이용자 131만. 이번 조사에서 거지맵의 성균관대·명륜동 관련 언급을 재검색했으나 신규 자연 발화 후기는 확인되지 않음(ncli blog 질의 0건) — **성대 상권에서의 거지맵 커버리지·인지도는 여전히 미검증**(선행 리서치와 동일 한계, 재확인만).

### 1-4. 지도 기반 메뉴 가격 서비스 — 공식 채널 현황 재확인

- 성균관대 공식: `welfare_11.do`(인사캠 식당/메뉴) + KINGO-M 앱. KINGO-M은 학사정보·포털 통합 앱이며 식당 메뉴는 부가 기능 — 출처: [성균관대 KINGO-M 앱 - App Store](https://apps.apple.com/kr/app/%EC%84%B1%EA%B7%A0%EA%B4%80%EB%8C%80-kingo-m/id1387421861) — 접근 2026-07-19. 기존 리서치가 이미 "메뉴·가격 콘텐츠 빈약"으로 확인한 채널과 동일.

## 2. 메뉴판 사진 → 구조화 가격 데이터: 유사 사례가 있는가 (신규 조사, 핵심 질문)

- **국내 소비자 대상 서비스는 확인되지 않았다.** 검색 범위: OCR 서비스 일반, 메뉴 OCR 특정.
- 유일하게 발견된 유사 개념: **MenuOCR** (글로벌 B2B 도구) — "팀이 카메라로 물리적/디지털 메뉴를 촬영하면 OCR로 항목·가격·카테고리를 자동 식별해 BI/CRM/ERP/스프레드시트로 흘려보낸다"는 경쟁분석용 데이터 수집 도구. 소비자용 추천 서비스가 아니라 **기업의 가격 모니터링 백엔드**다.
  - 출처: [MenuOCR - Apps on Google Play](https://play.google.com/store/apps/details?id=com.menuocr.consumer) — 접근 2026-07-19.
- 국내 OCR 인프라(Naver CLOVA OCR, NHN Cloud OCR)는 한국어 인쇄체 인식 정확도 95~97%로 존재하나, "메뉴판 사진 → 예산 기반 학생 대상 추천"에 적용한 사례는 검색상 확인되지 않음.
  - 출처: [Naver Clova OCR](https://www.ncloud.com/product/aiService/ocr), [한국어 OCR 솔루션 8가지 - Lido](https://www.lido.app/kr/hangugeo-ocr) — 접근 2026-07-19.
- **결론**: "사진 한 장 → 자동 구조화 → 크라우드소싱 데이터 파이프라인"을 대학가 식당 추천에 적용하는 조합은 이번 조사 범위에서 발견되지 않았다. 거지맵조차 수동 크라우드소싱이라는 점(선행 리서치 확인)과 결합하면, 이 축이 실제로 비어 있는 지대일 가능성이 높다 — 단, "검색으로 못 찾음"이 "존재하지 않음"의 증명은 아니라는 한계는 명시(비공개/스텔스 서비스 가능성 배제 못함).

## 3. "그냥 ChatGPT에 물어보면 되지 않나" — 강화 근거 (신규 조사)

일반론적 환각 통계는 확보했으나, "학식/식당 가격" 도메인에 특화된 1차 사례는 검색에서 직접 확보하지 못했다(한계로 명시). 대신 아래 근거로 논증을 구성한다.

- **환각률 정량 근거**: OpenAI 자체 벤치마크(PersonQA)에서 o3 모델 33%, o4-mini 48% 환각률 — 최신 모델도 사실 질의에서 상당한 오류율을 보인다는 공신력 있는 1차 근거.
  - 출처: [파이낸셜뉴스 "믿었던 챗GPT가 환각률 48%라니"](https://www.fnnews.com/news/202504201250325892) — 접근 2026-07-19.
- **구조적 이유(선행 리서치 `product-conflict-gate.md` §3에서 이미 정리, 재인용)**: (1) 오늘 날짜 기준 실시간 학식 메뉴를 모른다 (2) 실제 현재 가격을 모른다 — 숫자는 환각 취약 영역 (3) 영업 여부·브레이크타임을 실시간 반영 못한다 (4) GPS 기반 거리 계산을 못한다.
- **왜 이게 학식·식당 가격 도메인에서 특히 치명적인가(신규 논증)**: 학식 메뉴는 요일 단위로 매일 바뀌고, 학교 홈페이지는 로그인 없이 접근 가능하지만 범용 LLM의 학습 데이터에는 포함되지 않는(또는 stale한) 실시간 표다. 식당 메뉴판 가격은 애초에 텍스트 웹에 존재하지 않고 사진 안에만 있다(§2, `source-feasibility.md` 확인) — 즉 ChatGPT가 "모르는 정도"가 아니라 "학습 데이터 자체에 원천적으로 없는" 영역이다. 이는 일반적 환각(오답)보다 근본적인 공백(무입력)이라는 점에서 더 강한 차별화 논거가 된다.
- **거지맵과의 차이 재확인**: 거지맵도 크라우드소싱 방식이라 ChatGPT보다는 낫지만 수동 입력 비용 때문에 커버리지가 제한적 — 이 지점이 "범용 LLM 대비"가 아니라 "기존 서비스 대비" 차별화 축(사진→자동 파싱)으로 이어진다 (선행 리서치 ADR-0001과 일치).

## 4. 명륜동·혜화 식당 정보의 기존 공개 소스 현황 (신규 조사)

- ncli blog 검색("명륜동 혜화 대학가 식당 가격 지도")에서 상위 노출은 전부 **개별 맛집 후기 블로그 포스트**(분위기·감상 중심 산문, 2016~2025년 산발적)였다 — 구조화된 가격 비교표·지도형 콘텐츠는 검색 상위에 없음.
  - 확인된 예: [명륜동식당 후기](https://blog.naver.com/b78nzcbjdod4z/224091317142)(2025-11-28), [혜화맛집 명륜동식당](https://blog.naver.com/jx5zxo05e4n/224112118020)(2025-12-16), [성균관대 팔방미인 맛집](https://blog.naver.com/lu2u3u/220918055669)(2017-01-23), [명신식당 순대국밥](https://blog.naver.com/lu2u3u/220815333574)(2016-09-19) — 접근 2026-07-19.
- 이는 선행 리서치 `source-feasibility.md`의 "네이버 Local API에 가격 필드 없음(S7 실행 확인)" + "메뉴판 사진 안에 갇혀 있을 가능성"과 정합적이다 — **명륜동/혜화 상권의 식당 가격 정보는 산문형 블로그와 사진 속에 흩어져 있을 뿐, 구조화된 1차 소스가 공개돼 있지 않다.** 이번 조사는 이 결론을 뒷받침하는 신규 표본(4건)을 추가했을 뿐, 구조화 소스를 새로 발견하지는 못했다.
- 성대 커뮤니티(에브리타임 등)발 수작업 식당지도 사례는 선행 리서치가 고려대(안암) 사례(S-N01b)로 확인했으나, 성균관대 특정 사례는 미확보 상태 그대로다(추가 검색에서도 신규 확보 실패).

## 5. 조사 한계

- Naver 색인 기반 검색(ncli)과 WebSearch는 **에브리타임·카카오톡 오픈채팅 등 폐쇄형 채널을 커버하지 못한다** — 선행 리서치와 동일한 사각지대가 이번 조사에도 그대로 적용된다.
- "메뉴판 사진 구조화 서비스 없음"은 공개 검색 결과 기준이며, 비공개/베타 단계 서비스의 존재를 배제하지 않는다.
- ChatGPT 환각의 "학식/식당 가격" 도메인 특화 1차 사례(예: 실제 사용자가 ChatGPT에 학식 가격을 물어 틀린 답을 받은 캡처)는 이번 조사에서 직접 확보하지 못했다 — 발표에서는 일반 환각 통계 + 구조적 논증(§3)으로 대체해야 하며, "구체 사례 있음"으로 과장하면 안 된다.

## 6. 차별화 서사에 쓸 핵심 논거 3개

1. **"메뉴판 사진 → 자동 구조화" 조합은 국내 소비자 대상 대학가 식당 추천 서비스에서 실사례가 확인되지 않는다.** 유사 개념(MenuOCR)은 해외 B2B 가격 모니터링 도구일 뿐, 학생 대상 예산 추천에 쓰인 사례는 없다(§2). 거지맵을 포함한 기존 서비스는 전부 수동 크라우드소싱(선행 리서치 확인) — 데이터 생산 비용을 낮추는 축은 이번 조사로도 비어 있는 지대로 재확인됐다.
2. **ChatGPT의 한계는 "오답"이 아니라 "원천 부재"다.** 학식 메뉴(매일 갱신)와 주변 식당 가격(애초에 텍스트 웹에 없고 사진 속에만 존재, §4)은 범용 LLM 학습 데이터에 구조적으로 들어갈 수 없는 정보다. 여기에 최신 모델(o4-mini)도 사실 질의 환각률 48%라는 1차 근거(OpenAI 벤치마크)를 더하면, "그냥 물어보면 되지 않나"에 대한 답은 "모델 성능 문제가 아니라 정보 자체가 없다"로 정리된다.
3. **학식 전용 앱은 대학마다 반복 재생산되며(서울대만 학식샤맵+식샤 2종, 숭실대 EAT-SSU) 이미 "안 풀린 문제"가 아니다** — 차별화는 "학식 정보를 준다"는 존재 자체가 아니라, 학식과 주변 식당을 예산 제약 아래 **한 판에서 자연어로** 엮는 조합, 그리고 그 데이터를 채우는 파이프라인(사진→파싱)에 있다. 캠퍼스픽은 이 경쟁군에서 애초에 제외해야 한다(§1-1 오해 정정) — 발표 자료에 잘못 넣으면 사실관계 오류로 감점 리스크.
