# 메뉴판 파싱 PoC (M2) — 실행 안내

> 목적: "메뉴판 사진 → Document Parse → 메뉴·가격 자동 추출"이 실제로 되는지, 정확도가 몇 %인지 확인.
> 이 결과가 프로젝트 차별화 축의 검증이자 발표 자료의 실측 근거가 된다.

## 준비물
- Upstage API 키 (console.upstage.ai → API Keys, `up_`로 시작)
- 명륜동 실제 식당 메뉴판 사진 3~5장 (폰 카메라로 찍은 것 그대로 — 일부러 깨끗한 사진만 고르지 말 것. 흐릿한 것·비스듬한 것도 섞어야 실측이 됨)
- Node.js 18+

## 실행

```bash
# PowerShell
$env:UPSTAGE_API_KEY="up_xxx"
node scripts/parse-menu-poc.mjs 사진1.jpg 사진2.jpg 사진3.jpg
```

사진 1장당 $0.01 (가입 크레딧 $10 = 1,000장이라 부담 없음).

## 결과 기록 (accuracy.md)

각 사진마다 실제 메뉴판과 출력(`*.menu.json`)을 눈으로 대조해 아래 표를 채운다:

| 사진 | 실제 메뉴 항목 수 | 정확 추출(이름+가격 모두 일치) | 이름만 틀림 | 가격만 틀림 | 누락 | 비고(사진 상태) |
|---|---|---|---|---|---|---|
| ex) bunsik1.jpg | 24 | 21 | 1 | 0 | 2 | 정면, 조명 양호 |

정확도 = 정확 추출 ÷ 실제 항목 수. PRD 목표: ≥90%.

## 판정 기준
- **≥90%**: 가설 검증 성공 → api/parse-menu.js 본구현 진행
- **70~90%**: 사용자 수정 전제로 진행 가능 — "파싱 결과를 폼에 미리 채우고 사용자가 고침" 설계가 필수임을 확인한 것
- **<70%**: `ocr: force` 외 옵션 조정 실험 후 재측정. 그래도 낮으면 멘토링 질문 2번(파싱 신뢰 경계)에 실측치를 들고 가서 물어볼 것 — 실패 데이터도 산출물이다.
