# DR11 — 설계도와 화면을 사실로 되돌린다

> 완료 2026-07-22 · Horizon `demoday-run` · plan `plans/2026-07-21-dr11-design-truth.md`
> 감사 근거 `research/2026-07-21-dr11-design-doc-audit.md` · 검증 `verification/matrix.md` #133~151

## 1. 문제 정의

사용자 지시는 둘이었다 — "전체 설계 문서가 어디 있나, 채워야 할 부분을 알아보고 새 계획을 잡자"
그리고 "페이지별 UI 도 손봐야겠다"(지도 카드 잘림 · 학식 9 라벨 · 랜딩 아랫부분 · 제보 · Upstage 절).

조사해 보니 흩어진 요청이 아니라 **한 가지 병의 증상들**이었다. 2026-07-19~21 에 코드를 여섯 번
크게 고치는 동안(개인화 재정의 · 대화 칩 · 학식 정본화 · 마커 카드 · TRD 개정) **설계 문서를 한 번도
안 고쳤다.** 그 결과:

- **라이브 `/about.html` 이 어제 지운 기능을 광고**하고 있었다(ADR-0004 가 폐기한 순서 개인화).
- `docs/SITEMAP.md` 가 6곳에서 구현과 어긋나고 **자기 안에서도 모순**됐다.
- 문구·라벨에 규약이 없어 "학식 9"(전 날짜 누적 행 수)가 근거 없이 남았다 — cron 이 도는 동안
  계속 불어나는 숫자였다.

## 2. Objective 연결

- **심사 가능한 완성도**: 7/23 미팅에서 다른 버전과 나란히 놓고 비교한다. 심사위원이 about 을 읽고
  app 을 보면 즉시 어긋나는 상태로는 갈 수 없었다.
- **처리 깊이·검증 가시성**: 발표 준비 때 설계도를 읽으려면 그 설계도가 사실이어야 한다.

## 5. 이슈와 해결 — 검증이 계획을 두 번 구했다

**① 감사가 두 페이지의 진단을 서로 바꿔 놓았다.** 계획은 "랜딩 데모는 보정이 없어 항상 잘리고,
`/app.html` 은 보정이 있어 정상"이라고 적었다. **둘 다 틀렸다.** 실측하니 랜딩은 마커 22개 전수
잘림 0(초기 `setBounds` 여백이 마커를 가장자리에서 떼어놓는다), `/app.html` 은 **상단 270.5px 넘침**
— 카드 높이(251px)보다 커서 통째로 안 보였다.

진짜 원인은 "보정이 없다"가 아니라 **"보정이 느리다"**였다. `panBy` 가 애니메이션이라 결국 카드는
들어오지만 그 전까지 1초 가까이 잘려 보인다. 사용자가 본 건 그 1초다. **코드에 보정이 있다는 사실은
정상의 증거가 아니다** — 사용자가 보는 시점(클릭 320ms)의 상태를 재야 잡힌다.

고치는 과정에서 손잡이를 두 번 잘못 골랐다: `yAnchor` 는 오버레이 생성 시점에 고정이라 높이를 알기
전엔 못 정하고, CSS 클래스 transform 은 `.in` 전환과 충돌해 아예 안 먹었다(배포 후 computed 값으로
확인). 카카오가 앵커를 구현하는 실제 수단인 **컨테이너 inline margin** 이 유일하게 확실했다.

**② 감사가 놓친 잔재를 Verify 가 잡았다.** step-1 의 `grep -n "순서"` 가 세 번째 위치를 찾아냈다 —
로그인 다이얼로그의 같은 주장이 `about`·`app`·`index`·`verify` **4페이지에 복제**돼 있었다. 감사는
페이지 본문만 훑고 공통 조각을 안 봤다. 이 사실을 SITEMAP §3 과 두 agent 브리프에 규약으로 박았다.

**③ 부수 수확 — cron 첫 자동 실행 관측.** 어제 남은 유일한 미관측 항목이었다. `/api/data` 의
`today=2026-07-22` 에 패컬티식당 당일 식단이 있었다 — **사람 손 없이 갱신되는 것이 처음 확인**됐다.

## 6. 결과물과 증거

**실표면**: 배포 URL https://sixsense.askewly.com — 실브라우저(Playwright, 1440×900 · 390×844)로
5페이지를 열어 카운터·마커 카드·제보 폴백을 직접 조작해 관측했다.

**재현**:
```
node scripts/demo-smoke.mjs --url https://sixsense.askewly.com   # 14/14 PASS
node scripts/test-chat-extract.mjs      # 35/35
node scripts/test-side-menu.mjs         # 41/41
node scripts/test-personalization.mjs   # PASS (ADR-0004 경계)
node scripts/test-chips.mjs             # 18/18
```

**핵심 관측**(전문 → `verification/matrix.md` #133~151):
- `/app.html` 마커 14개 전수 **잘림 0**. 최악 케이스 **+270.5px → -0.7px**
- 클릭 **25ms** 시점에 이미 경계 안(-86) — 애니메이션 이벤트에 비의존
- 카운터 `24 / 342 / 1` · 라벨 `오늘 문 연 학식당` — `/api/data` `cafeteriaBoard` 와 일치
- 제보 0행 경로: 1단계로 안 튕기고 수기 입력 제공. `"8,000원"` → `8000` 수집. **시트 미제출**
- 5페이지 죽은 링크 **0**(앵커 포함) · 순서-개인화 주장 **0건** · 콘솔 red **0**
- 배포 반영 probe: 이번 변경 문자열 9개 전부 배포본에 존재

**한계(정직 기록)**: 학식당 0곳인 날의 "휴무" 표시는 오늘 1곳이 열려 **실측하지 못했다**. 코드 분기만
확인했다. 다음 주말·방학일에 확인이 필요하다.
