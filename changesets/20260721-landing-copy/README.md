## 카운터 — "학식 9" 의 정체

```js
document.getElementById("nCaf").textContent = d.cafeteria.length;   // 구
```

`d.cafeteria` 는 `[학식]` 시트의 **전 날짜 누적 행**이다. 학식당 수도 오늘 메뉴 수도 아니고,
어제 켠 cron 이 매일 도는 동안 **계속 불어나는 숫자**였다(며칠 뒤 "학식 40").
오늘로 거른 값은 `api/data.js` 가 이미 `cafeteriaBoard` 로 따로 내려주고 있었는데 쓰지 않았다.

셋 다 규약(§SITEMAP 5)에 맞춰 라벨을 고쳤다:

| 구 | 신 | 값의 실제 의미 |
|---|---|---|
| 식당 | **수집한 식당** | 검수 통과 메뉴가 1개 이상인 식당 |
| 메뉴 | **검수 통과 메뉴** | `검수="확인"` 행 |
| 학식 | **오늘 문 연 학식당** | `cafeteriaBoard.filter(open)` |

**0곳일 때는 숫자 대신 "휴무"** 를 쓴다(방학·주말). `0` 은 사실이지만 "학식이 없어진 서비스"로 읽힌다.

## 아랫부분 문구

- 카운터 아래 주석 한 줄 신설(죽어 있던 `.proof-note` 스타일을 씀 — 신규 토큰 없음):
  "가격은 메뉴판 사진에서 뽑아 사람이 검수한 것만 셉니다. 학식은 매일 아침 6시에 자동으로 받아옵니다."
- `band-ink` 에 리드 신설: **"학식은 이 길을 지나지 않습니다"** — 이미 표로 정리된 HTML 이라 크롤링하고,
  같은 도구를 억지로 쓰지 않는 것도 설계다(ADR-0001). 랜딩 아랫부분에 없던 **두 번째 수집 경로**가
  이걸로 드러난다. 파이프라인 20점 서사에서 빠져 있던 축이다.
- 3단계 설명 보강: "추천 이유가 데이터에 근거했는지 **다른 모델**이 판정합니다" — 생성자와 판정자를
  분리한 것이 이 서비스의 핵심인데 랜딩에서는 그냥 "판정합니다"로 뭉개져 있었다.
