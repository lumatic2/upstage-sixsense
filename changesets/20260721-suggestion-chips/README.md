# 20260721-suggestion-chips

- Target: 사용자 지시 2026-07-21 — "칩의 존재 이유는 에이전트와 대화를 쉽게 하기 위함이다.
  클릭만으로도 대화가 계속되게, 칩 3개씩 대화 맥락에 맞게 생성하라."
  선택지 3안 제시 후 **① Solar 생성 + 서버 검증 ② 첫 화면도 맥락 칩** 확정.
- Scope: `api/chat.js`(스키마·프롬프트·조건 인계), `api/_lib/chips.js`(신규 검증), `api/_lib/geo.js`(신규 — 도보 환산 공용화),
  `api/recommend.js`(지역 하버사인 → `_lib/geo.js`), `public/app.html`(칩 렌더·진입 칩·`.chips` 줄 제거), `scripts/test-chips.mjs`(신규).
- 문제 정의:
  - 입력창 위 고정 예산 칩 줄(6천/8천/1만/전체)은 **첫 질문 뒤에도 그대로 남아** 이미 답한 질문을 계속 권했고,
    무엇보다 대화 맥락과 무관한 고정 선택지라 "대화를 쉽게 한다"는 칩의 존재 이유를 못 하고 있었다.
  - 그렇다고 칩 문장을 Solar 에 자유롭게 맡기면 "24시 하는 곳"·"배달되는 곳" 처럼 **데이터에 없는 제안**이 나온다.
    누르면 그 조건과 무관한 결과가 나오고, 그건 이 서비스가 계속 막아 온 결함(말과 화면이 다르다)이다.
    반대로 코드가 조건 슬롯만 보고 고르면 "맥락에 맞는 제안"이 아니라 빈칸 채우기 —
    DR9 에서 걷어낸 "에이전트인 척하는 if/else" 로 되돌아간다.
- Contract — **문장은 모델이 쓰고, 진실 여부는 데이터가 정한다**:
  - Solar 가 후보를 5개까지 낸다(`chips` 구조화 출력). 서버가 실데이터로 돌려보고 살아남은 **최대 3개**만 내려보낸다.
  - 검증 항목(`api/_lib/chips.js`): 예산 → 그 금액에 **본메뉴**가 있는 집이 1곳 이상(곁들임만 남는 집은 한 끼가 아니다) ·
    도보 → 상한 안에 실제로 1곳 이상(`/api/recommend` 는 전부 걸러지면 상한을 완화하므로 검증 없이는 "도보 3분 안"이 거짓이 된다) ·
    태그 → 데이터에 실제 있는 어휘이고 그 태그를 가진 집이 1곳 이상 · 지금 조건과 같으면 버린다.
  - 칩 클릭은 **대화 한 턴**이다(라벨이 그대로 사용자 발화가 된다). 조건은 이미 검증됐으므로
    `/api/chat` 에 `conditions` 로 실어 보내고 서버는 문장에서 다시 뽑지 않는다 —
    다시 뽑으면 "도보 5분 안으로" 라벨에서 예산이 조용히 풀린다.
  - 칩은 **덧붙이는 조건**이다. 태그는 누적하고 금액·거리는 덮어쓴다(`mergeConditions`, 서버·클라이언트 동일 규칙).
  - 진입 칩은 대화가 없어 맥락도 없다 → **오늘 데이터**에서 뽑는다(실제로 결과가 나오는 예산, 오늘 문 연 학식).
    후보를 지어내지 않고 셈으로 거르므로 같은 계약이다. 추가 왕복 없음(이미 받은 `/api/data` 로 계산).
    **갈래를 하나씩 섞는다 — 학식 / 예산 / 상황**(사용자 지시 2026-07-21). 예산 칩 두 개를 늘어놓으면
    "예산으로만 묻는 서비스" 처럼 읽힌다. 상황 칩은 데이터에 실제 있는 태그에서만 고른다(혼밥 → 가성비 → 단체 순).
- 검증 중 적발해 막은 것 (실측에서 Solar 가 실제로 낸 불량):
  - **라벨-조건 불일치** — `"한식만 보기"` 에 `생선` 태그를 실어 왔다. 누르면 라벨이 약속한 것과 다른 게 나온다.
    → 태그가 라벨에 드러나지 않으면 버린다.
  - **예산 몰래 올리기** — 8천원 조건에서 `"학식 보기"` + `budget:9000` 을 냈다. 사용자가 말한 적 없는 인상이다.
    → 금액을 올리는 칩은 라벨에 숫자가 보일 때만 통과.
- 남는 한계(의도적):
  - 검증을 통과하는 후보가 3개 미만이면 **그만큼만** 뜬다(1~2개). 숫자를 맞추려고 못 미더운 후보를 살리지 않는다.
    (데이터에서 계산한 칩으로 빈자리를 메우는 안을 제시했으나 **"꼭 3개일 필요는 없다"** 로 확정 — 사용자, 2026-07-21.)
  - 태그가 라벨에 없으면 버리므로 "혼자 가기 좋은 곳"(=혼밥) 같은 자연스러운 표현도 탈락한다. 지어낸 제안보다 낫다.
