## 문서 실사에서 나온 것

개인화는 **어느 설계 문서에도 없었다.** `OBJECTIVE.md`·`PRD.md`·`ARCHITECTURE.md` 에 언급이 없고,
`docs/TRD.md` §5.7 의 개인화("최근 검색어 5개 저장")는 **팀원 원본 데모(hanipmap)의 문서**다
(같은 문서가 Naver Maps·Supabase·영업시간 판별을 설명한다 — 이 레포에 없는 것들이다).
즉 돌아가던 개인화는 changeset 에만 기록돼 있었고, `CAT_BONUS=60` 이 근거를 못 댄 것은
값의 문제가 아니라 **정의가 없었기 때문**이었다.

## 결정 ① — 순서 개인화(분류 관심 가점)를 걷어낸다

신호가 허수였다. 이 서비스에는 "선택"이라는 사용자 행동이 없다:

- 추천 카드·학식 카드는 클릭을 받고 있었지만 **눌러도 아무 일도 일어나지 않았다**
  (`.card.liked` 는 커서 모양과 hover 테두리뿐). 누를 이유가 없는 클릭으로 취향을 추론했다.
- 반응이 있는 클릭은 지도 마커뿐인데, 그건 "이 가게가 궁금하다"보다 "지도에서 눌러봤다"에 가깝다.

즉 **일어나지 않은 선택을 근거로 순서를 바꾸고 카드에 배지까지 달고 있었다.** 값을 고치는 대신
축을 닫는다. 제거: `cats` 수집 · `CAT_BONUS` · `readProfile` 가점 · `personalized` 플래그 ·
"전에 본 곳과 같은 분류" 배지 · 해명 턴의 분류 요약/`반영 끄기` · 반응 없는 `.card.liked`.

이제 `/api/recommend` 는 프로필을 **받지도 읽지도 않는다.** 순서는 명시 조건(태그·예산·도보)만으로
정해지므로 "왜 이 집이 위인가"에 항상 답할 수 있다.

## 결정 ② — 개인화 = 대화 연속성

> 지난 방문에 물었던 관심을 이어받아 **답한다**. 후보도 순서도 바꾸지 않는다.

**기억 대상 = 데이터로 검증된 관심어.** 발화 원문도 모델 요약도 저장하지 않는다.

```
사용자 문장 → 어절 분리 → 조사 제거 → 메뉴명·식당명·분류와 대조 → 매칭 ≥1 이면 저장
저장: { foods: [{w:"돈까스", at:…}], last: {…} }   ← 브라우저에만
```

- **0건이면 저장 안 함** — "마라탕버거"는 기억하지 않는다. 다음 방문에 그 말을 꺼내면 헛말이다.
- **부분일치 금지** — 사용자가 말한 낱말 **그대로**가 데이터에 있어야 한다. 반대 방향으로 잡으면
  "마라탕버거"에서 `마라탕` 을 건져내게 된다(이슈 ⑧ 과 같은 함정).
- **꺼낼 때마다 재검증** — 기억은 브라우저에 남지만 데이터는 바뀐다. 사라진 메뉴는 말하지 않는다.
- 보관: 관심어 5개 · 30일 · `localStorage` 만. 서버는 받아 쓰고 버린다(무상태).

**쓰는 곳 둘**: 재방문 첫 턴(관심어+조건을 한 덩어리로, 그 자리에서 지울 수 있게) ·
이후 모든 턴에 `<지난방문>` 주입(문장은 모델이 쓴다).

## 검증 중 적발해 고친 것

- **없는 기억을 지어냈다.** `<지난방문>` 이 **없는데도** "지난번엔 일식 찾으셨는데…" 를 냈다(실측).
  규칙 9 를 프롬프트로 강하게 쓴 대가다. 프롬프트로 다시 막으면 다음 문형에서 또 새므로
  **코드로 막았다** — `stripFabricatedMemory()`: 기억을 말하는 문장은 검증된 관심어를 담아야 통과.
  (예산 문장에서 이미 같은 실패를 겪었고 같은 해법을 썼다.)
- **재방문 인사가 두 덩어리로 떴다.** "지난번엔 이런 걸 물어보셨어요"(관심어) 와
  "지난번에는 이렇게 보셨어요"(조건)가 따로 나와 같은 말이 반복됐다 → 한 턴으로 합쳤다.
- **검증 자체가 낡은 서버를 보고 있었다.** 로컬 서버가 포트에 물려 있어 프롬프트 수정 3회가
  전부 **옛 코드**로 측정됐다("고쳐도 안 된다"의 정체). 포트를 확실히 끊고 다시 재니 한 번에 통과.

## 곁들여 넣은 것 — 카카오맵 길찾기

추천 카드·마커 카드에 좌표를 넘기는 링크 하나. 새 데이터도 API 키도 필요 없다(이미 `lat`/`lng` 보유).
"찾았으면 갈 수 있어야 한다" — 개인화와 무관한 실용성 항목이다.
**이 클릭은 기록하지 않는다**(ADR-0004 §2) — 근거가 생겼다고 바로 재는 것이 이 사고의 재발 경로다.
