# PRD — 한입지도 (팀 sixsense)

> 2026-07-15 확정. 결정 근거: `docs/adr/0001-problem-and-solution-shape.md`, 리서치: `docs/research/`.
> 대회 제약·루브릭: `docs/CHALLENGE.md`. 팀원 데모: github.com/ljb0138/hanipmap

## 목표
성균관대 학생이 예산·상황을 문장으로 말하면, 학식과 학교 주변 식당을 한 판에 놓고 지금 갈 수 있는 곳을 추천한다.

## 사용자
- P1: 성균관대 재학생(명륜캠 우선). 한 끼 예산이 정해져 있고, 학식·주변 식당 사이에서 매번 고민. "교수님과 식사", "시험기간 24시" 같은 상황 제약이 자주 붙는다.

## 핵심 기능 (MVP)
1. **자연어 조건 검색**: "8천원 이하, 도보 5분, 혼밥" → Solar가 구조화(예산·거리·태그) → 필터+지도 추천. (데모의 정규식 파싱을 Solar로 교체)
2. **사진 제보 파이프라인** ★차별화 축: 메뉴판 사진 업로드 → Upstage Document Parse → 메뉴·가격 자동 구조화 → 검수 → DB 반영. 제보 비용을 "타이핑 수 분"에서 "사진 1장"으로.
3. **학식 통합**: 성대 학식 페이지(HTML 확인됨) 크롤링 → 오늘의 학식이 식당과 같은 추천 풀에 등장.
4. **예산 조합 추천 이유**: 예산 안 최대 메뉴 조합 계산(데모 기존 로직) + Solar 생성 추천 이유.
5. **개인화 = 대화 연속성 하나**: 지난 방문에 물었던 관심(예: 돈까스)을 이어받아 답한다 — 단 **실제 데이터와 대조를 통과한 것만** 기억하고 말한다. **추천 순서는 개인화하지 않는다**(순서는 명시 조건만으로 결정 — 순서 가점 축은 신호가 허수라 폐기). 프로필은 브라우저에만 둔다(서버 무상태). 정의·경계는 **ADR-0004 가 정본**이고 이 줄은 그것을 인용한다.

## 범용 챗봇·기존 제품과의 차별점 (확정 답변)
- **vs ChatGPT/Claude**: 오늘자 학식 메뉴·실제 현재 가격·영업/브레이크타임을 모르고 숫자 환각이 있다. 우리는 1차 소스(학식 페이지 크롤링 + 파싱 검증된 메뉴판)만 답에 쓴다.
- **vs 거지맵(직접 경쟁, 리서치 확인)**: 거지맵의 데이터는 수동 크라우드소싱. 우리는 **사진→파싱 파이프라인으로 데이터 생산 비용 자체를 낮춘다** + 학식 크롤링 자동 통합 + 자연어 상황 검색. 발표에서 이 대비를 숨기지 말고 정면으로 다룬다.

## MVP 제외 사항
- 영수증 파싱·잔여 예산 상태 관리 — 범위 초과(7/24 마감). Next Candidate 보존.
- 자연캠(율전) — 명륜캠 검증 후.
- 리뷰·평점·커뮤니티 기능 — 에브리타임과 정면 충돌 지대.

## 성공 지표
- 메뉴판 사진 → 메뉴·가격 추출 정확도: 실제 명륜동 메뉴판 N장 골든셋 기준 항목 단위 ≥90% (PoC에서 측정·확정)
- 학식 크롤러: 평일 식단 자동 수집 성공
- E2E: 사진 제보 → 파싱 → 승인 → 추천 노출까지 데모 시연 가능

## Upstage 제품 사용 (하드 요건)
- **Document Parse**: 메뉴판 사진(JPEG/PNG) → 구조화. 학식에는 사용 안 함(이미 HTML — 억지 적용 금지, ADR-0001).
- **Solar**: 자연어 질의 → JSON 구조화, 추천 이유 생성.
