# 시연 영상 촬영 순서 (5스텝)

> **누가**: 영상은 사용자(전유성)가 화면 녹화로 찍는다. 발표는 팀원이 하고, 영상은 발표 7분 안에서 재생된다.
> **어디서**: 반드시 **라이브 URL** `https://sixsense.askewly.com` 에서 찍는다(제출하는 Live URL과 같아야 한다). 개발 서버(localhost)로 찍지 않는다.
> **내레이션**: 팀원이 재생 중 말로 설명하면 **무음**으로 찍어도 된다. 영상만 단독으로 돈다면 **화면에 자막**을 얹는다 — 팀원과 먼저 맞춘다.
> **길이 목표**: 60~90초. 아래 5스텝을 또박또박, 핵심 화면에서 2~3초 멈춰 읽을 시간을 준다.

## 원칙 — 랜덤 클릭 금지

심사 55점이 데이터 처리·Upstage에 걸려 있다. 그냥 클릭만 하면 "지도 앱"으로 보인다.
**근거가 화면에 남는 순서**로 움직인다. 아래 5스텝이 그 순서다.

---

## 스텝 1 — 랜딩: "챗봇은 못 하는 일" (10초)

- 열기: `https://sixsense.askewly.com/`
- 상단 히어로에서 예산 금액이 타이핑되는 것을 잠깐 보여준다.
- 아래로 스크롤 → **듀오 카드**("한입지도는 메뉴를 알아요") 에서 2~3초 멈춘다.
  - 왼쪽(범용 챗봇): *"지금 계신 위치를 몰라서요… 직접 확인해 보세요"*
  - 오른쪽(한입지도): *"경영관에서 도보 N분 · [식당] / [메뉴] [가격]"*
- **메시지**: 같은 질문에 챗봇은 못 답하고, 한입지도는 실데이터로 답한다.

## 스텝 2 — 메뉴 찾기: 추천 + 근거 검증 (20초, 가장 중요)

- 상단 메뉴 **"메뉴 찾기"** 클릭 (또는 랜딩 "메뉴 찾아보기").
- 예산 입력창에 자연어로 친다: 예) **`8천원 이하 10분 거리 혼밥`**
- 잠시 후 추천 카드 3개가 뜬다. 여기서 **꼭 보여줄 두 가지**:
  1. 각 카드의 **추천 이유 한 문장** (Solar가 실데이터로 쓴 것)
  2. 대화 요약줄: **"추천 이유는 다른 모델이 데이터와 대조해 N건이 근거 검증을 통과했습니다"**
     ← **이 줄이 근거 검증의 항상 보이는 증거다.** 3초 멈춘다.

> ### ⚠ "근거 미달 → 데이터 교체" 배지에 대하여 (읽어둘 것)
> 이 배지는 **아무 때나 안 뜬다.** solar-pro2가 근거 없는 표현(맛·평판·분위기)을 쓴 걸
> solar-mini가 잡아냈을 때**만** 붙는다. 그런데 생성 프롬프트가 그런 표현을 애초에 막아서
> **실제로는 거의 항상 통과한다**(2026-07-24 실측: 15개 추천 전부 통과, 교체 0건).
> → **영상에서 이 배지를 억지로 띄우려 하지 말 것.** 안 뜨는 게 정상이고, 오히려 "생성이
>   잘 통제돼 있다"는 증거다. 근거 검증은 **위의 요약줄**로 보여주면 충분하다.
> → "교체되는 장면"을 꼭 보이고 싶으면 **스텝 3의 `/about.html` 3단계 예시**(초안→판정→교체)를
>   쓴다. 거기엔 그 장면이 **항상** 그려져 있다.

## 스텝 3 — 데이터 파이프라인: 처리의 깊이 (20초)

- 상단 **"데이터 파이프라인"**(= `/about.html`) 클릭.
- 위에서 아래로 스크롤하며 **네 곳에서 멈춘다**:
  1. **파이프라인 흐름도**(입력 3갈래 → 검수 한 문 → 서비스) — 2초
  2. **파싱 정확도** 구간표(인쇄체 100% / 저해상 20%, 지배 변수는 해상도) — 3초
  3. **Upstage 4자리 표**(Document Parse + Solar 3자리, "학식엔 안 씀") — 3초
  4. **3단계 근거 검증 예시**(추천 이유 초안 → solar-mini 판정 "근거 없음" → 데이터 사실로 교체) — 3초
     ← 스텝 2에서 배지가 안 떠도, **여기서 교체 메커니즘을 확실히 보여준다.**
- (선택) 페이지 안의 **실행 블록**(예시 메뉴판 눌러 Document Parse 실제 호출, 문장 넣어 Solar 구조화)을
  하나만 눌러 "스크린샷이 아니라 진짜 API가 지금 돈다"를 보여주면 강하다.

## 스텝 4 — 사진 제보: 수집의 입구 (15초)

- 상단 **"사진 제보"**(= `/contribute.html`) 클릭.
- 메뉴판 사진 한 장을 올리는 장면 → **Document Parse가 그 자리에서 읽어** 메뉴·가격 후보를 뽑는 걸 보여준다.
- **메시지**: 데이터는 이렇게 **직접 만들어진다**. 아무나 한 장 올리면 파이프라인이 돈다.

## 스텝 5 — 운영자 검수: "사람이 통과시킨다" (10초)

- 페이지 맨 아래 **푸터 → "운영자 검수 →"**(= `/review.html`) 클릭.
- 비밀번호 입력(데모용) → 검수 화면: 파싱 결과를 원본 사진과 대조해 **확인/제외**로 가르는 자리.
- **메시지**: AI는 후보를 만들 뿐, **사람이 확인한 것만** 서비스에 나간다. 이게 "그냥 ChatGPT엔 없는" 자리다.
- (검수 화면은 시트 쓰기 권한이 있으니, 녹화 중 실제로 값은 바꾸지 말고 **화면만** 보여준다.)

## 촬영 체크리스트

- [ ] 라이브 URL(`sixsense.askewly.com`)에서 찍는가 (localhost 아님)
- [ ] 무음/자막 여부를 팀원과 맞췄는가
- [ ] 스텝 2의 **요약줄**("N건이 근거 검증을 통과")이 화면에 잡혔는가
- [ ] 스텝 3의 **4개 지점**(흐름도·정확도·Upstage 표·검증 예시)에서 멈췄는가
- [ ] 60~90초 안에 들어오는가
- [ ] 검수 화면에서 실제 값을 바꾸지 않았는가

## 자주 나오는 오해

| 오해 | 사실 |
|---|---|
| "근거 배지가 안 떠서 검증이 안 도나?" | 돈다. 통과하면 배지를 **일부러 안 붙인다**(장식 방지). 검증 결과는 카드의 `data-grounded` 와 요약줄에 남는다 |
| "클릭만 하면 되지?" | 절반만 맞다. **순서**가 중요하다 — 위 5스텝이 배점 표면을 지난다 |
| "localhost로 찍어도 되지?" | 안 된다. 제출 Live URL과 화면이 같아야 신뢰가 선다 |
