## 문제 — 명확한 타깃, 매일 반복되는 결정
<!-- 루브릭: 문제 정의의 구체성 및 타깃 명확성 (5) -->

- **타깃**: 성균관대 명륜캠 재학생. 한 끼 예산이 정해져 있고, 학식 ↔ 주변 식당 사이에서 매일 고민.
- 상황 제약이 늘 붙는다 — "교수님과 식사", "시험기간 새벽", "과선배가 사는 날".
- 핵심 장벽: **주변 식당의 실제 메뉴·가격은 어떤 공개 API에도 없다.**
  - 네이버 지역검색 API: 가격 필드 자체가 부재 (실행 확인)¹
  - 착한가격업소 CSV: 극소수 지정 업소만¹
  - 가격은 **메뉴판 사진 안에 갇혀 있다** — 우리가 실측으로 확인한 사실.

¹ `docs/research/2026-07-15-source-feasibility.md`

## 파이프라인 — 소스 3종, 성격에 맞는 3가지 처리
<!-- 루브릭: 정보 처리 파이프라인 설계 및 관리 (20) -->

```
[비정형] 메뉴판 사진 → /api/parse-menu (Document Parse)
            → {메뉴,가격} 후보 → 폼 프리필 → 사용자 수정 → pending → 운영자 승인 → DB
[정형]   성대 학식 HTML → 크롤러(direct GET, 매일) → cafeteria_menus → 추천 풀
[질의]   자연어 → /api/parse-query (Solar 구조화) → {예산·도보·태그} → 필터+조합 계산
```

- 오염 방지 3단: **자동 저장 금지** → 파싱 결과는 프리필만 → 사람 확인 → 승인.
- 수집(96곳) → 구조화 → 검수 → 서빙까지 전 단계가 레포에 코드로 존재.
- 스테이징 배포 URL에서 3종 전부 E2E 동작 검증³

³ `docs/research/2026-07-17-m3-e2e.md` (실브라우저 스크린샷 포함)

## 처리 깊이 — 정확도를 실측하고, 끌어올리고, 경계를 정했다
<!-- 루브릭: 정보 처리의 깊이 및 정교성 (15) -->

명륜동·혜화 실제 메뉴판 5장 골든셋, 항목 단위(이름+가격 모두 일치) 실측⁴:

| 표면 | 정확도 |
|---|---|
| 인쇄 책자 정면 (무제스시, 12항목) | **100%** |
| 유리창 세로 시트 (둘리분식, 25항목) | **100%** |
| 타일 카드·천원단위 표기 (18항목) | 66.7% |
| 손글씨 화이트보드 (7항목) | 57.1% |
| 저해상 웹 이미지 (10항목) | 20.0% |
| **종합 (72항목)** | **76.4%** |

- 후처리 반복 개선: **0건 → 62.5% → 76.4%** (OCR alt 승격 · 천원단위 `3.0` · `4 000` 공백 · 다항목 줄)
- 실패 모드에서 설계를 도출: 가격 오독(7.0→17.0) 실사례 → **전량 자동 반영 불가 → 검수 UX가 필수**라는 경계를 데이터로 확정.

⁴ `experiments/parse-poc/accuracy.md`

## Upstage — 목적에 맞는 곳에만, 실측 근거로
<!-- 루브릭: 목적에 적합한 제품 적용 (10) + 기술 적용 결과의 기여도 (10) -->

- **Document Parse** = 비정형(사진)에만. 학식 HTML에는 **의도적으로 미적용** — 이미 구조화된 데이터에 파서를 억지로 쓰지 않는 것도 설계다.
- **Solar(solar-pro2)** = 질의 구조화 + 추천 이유. "과선배가 사는 날 2만원으로 둘이" → `{budget:10000, tags:[격식]}` — 인원 나눗셈까지 실측 통과⁵.
- 운영 실측: 구조화 응답 0.7~1.3초, **2.5초 타임아웃 → 정규식 폴백** (데모 중 외부 장애에도 검색 불사). 모델 선정도 실측(solar-mini 예산 환각 관측 → pro2 채택)⁵.
- 기여도: 이 파이프라인이 없으면 제보 1건 = 수 분 타이핑. 지금은 **사진 1장 + $0.01**.

⁵ `changesets/20260717-parse-query-api/README.md`

## 실용성과 확장 — 오늘 배포돼 있고, 구조가 이식 가능하다
<!-- 루브릭: 실용성 및 활용 가능성 (10) + 서비스 논리 타당성 (5) + 임팩트·확장성 (5) -->

- 배포 데모: hanipmap.vercel.app (기존 데모) + 파이프라인 스테이징 E2E³.

- 확장: "학식 크롤러 + 메뉴판 파싱 + 예산 질의"는 대학가 단위로 복제 가능 — 서울대 학식도 동일 구조 HTML 확인¹.
