# 다이어그램 이미지 — 생성 기록

`/about.html` 이 쓰는 설명 그림이다. **값이 들어가는 그림은 여기 두지 않는다** —
정확도 막대·규모 수치처럼 바뀌는 숫자는 HTML/CSS 로 그린다(이미지로 구우면 다음에
숫자가 바뀔 때 아무도 못 고친다). 여기 있는 건 **구조만 담은 그림**이다.

## 왜 Codex 이미지 생성인가 (2026-07-23 실측)

처음엔 "생성 모델은 한글 라벨을 뭉갠다"고 보고 SVG 로 그릴 생각이었다. **틀렸다.**
시험 삼아 한 장 뽑아 보니 지시한 한국어 라벨이 **오타 0** 으로 나왔고 화살표 합류
구조도 지시대로였다. 예상으로 도구를 배제하지 말고 한 번 뽑아 보고 판단한다.

```bash
cd <임시 디렉터리>
codex exec --skip-git-repo-check -s workspace-write "<아래 프롬프트>"
```

### `pipeline-flow.jpg` — 파이프라인 전경

> 이미지 생성 도구로 이미지를 만들어 이 디렉터리에 flow-v2.png 로 저장해줘.
>
> 내용: 데이터 파이프라인 아키텍처 다이어그램. 왼쪽에서 오른쪽으로 흐름.
>
> 왼쪽 열 — '입력' 이라는 작은 제목 아래 박스 3개 세로 배치:
> '메뉴판 사진' / '학식 HTML' / '주소 텍스트'
>
> 가운데 열 — '처리' 라는 작은 제목 아래 박스 3개, 왼쪽 각 박스에서 화살표로 연결:
> 'Document Parse' / '크롤러' / '지오코딩'
>
> 오른쪽 — 세 처리 박스에서 화살표가 모두 하나로 합류해 '사람 검수' 박스로 들어가고,
> 거기서 화살표 하나가 '한입지도' 박스로 나감. '사람 검수' 박스와 '한입지도' 박스는
> 주황색 테두리로 강조.
>
> 색: 배경 #F7F9FC, 일반 박스는 흰 채움에 회색 테두리, 강조 박스는 #B4552D 테두리,
> 일반 화살표 회색, 마지막 화살표만 #B4552D. 글자는 진한 회색.
> 스타일: 깔끔한 플랫 다이어그램, 둥근 모서리, 얇은 선.
> 글자는 위에 적은 한국어 그대로 정확하게, 또렷하게 렌더링할 것. 그 외 다른 글자는 넣지 말 것.

**색은 프롬프트에 hex 로 박는다.** 안 적으면 밝은 주황(#F97316 계열)과 흰 배경으로 나와
사이트 팔레트(`--accent:#B4552D`, `--bg:#F7F9FC`)와 어긋난다 — 1차 시도에서 실제로 그랬다.

### 웹용 재인코딩

```bash
node scripts/encode-diagram.mjs <생성물.png> public/img/pipeline-flow.jpg 1600
```

765KB → 77KB (여백 절단 1774×887 → 1676×703). **PNG 로 다시 저장하면 오히려 커진다**(실측 1043KB) — 원본은 색이 몇 개뿐인
이미지인데 캔버스 리샘플링이 안티에일리어싱 계조를 잔뜩 만들기 때문이다. 자세한 판단은
`scripts/encode-diagram.mjs` 머리말에 있다.

생성 모델은 그림을 캔버스 가운데에 놓지 않는다 — 1차 생성물은 아래쪽 15%가 통째로 비어
본문에서 캡션과 벌어져 보였다. 재인코딩이 **배경색과 다른 픽셀의 경계를 재서 여백을 자동으로
자른다.** 뽑을 때마다 여백 위치가 달라지므로 눈으로 자르지 않는다.

## 그림을 고칠 때

라벨이나 구조가 바뀌면 **다시 뽑는다**(위 프롬프트를 고쳐서). 이미지 편집기로 글자만
덧칠하지 않는다 — 프롬프트와 결과가 갈라지면 다음 사람이 재현하지 못한다.
